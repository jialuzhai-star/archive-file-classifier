# Archive File Classifier

## Overview

提供一套“先盘点、后判断、再确认执行”的文件归档分类流程，适用于本地资料库整理、历史项目文档归档、混合办公文件批量分类等场景。

优先把风险控制放在前面：先只读分析、输出分类建议、等待确认，再执行备份与移动；默认不直接删除文件，只产出待删清单供用户自行处理。

## When To Use

在以下场景触发本 skill：

- 用户要求“整理文件夹”“归档资料”“按类别重分文件”
- 目标目录中包含大量 PDF、XLSX、XMind、DOCX、图片或混合文件
- 用户已经给出分类标准，或希望先共同沉淀分类标准
- 需要先识别边界文件、疑难文件、待确认文件，再批量移动
- 需要在高风险操作前自动执行备份、核对与复查

不在以下场景触发：

- 只是查看单个文件内容，不涉及归档结构调整
- 只是删除文件；删除属于高风险动作，默认只给清单，不代为执行
- 只是重命名单个文件，且不涉及批量分类流程

## Core Workflow

### Step 1: 锁定范围与分类标准

先确认以下信息；缺失时优先补齐，不要急着移动文件：

1. 目标目录是哪个
2. **选择或创建分类标准**：
   - **静默盘点**：先读取技能 `references/` 目录下现存的所有分类标准文件。
   - **新建标准**：如果泡芙大人明确要求“新建一套标准”，则暂停分类，进入【标准共建模式】，从0到1跟用户对话梳理出一套新规则，并保存为 `references/[新规则名称].md`。
   - **多选一询问**：如果 `references/` 目录下已有**多套**标准，必须先询问用户“本次归档使用哪套标准？”。
   - **默认规则**：如果用户未明确说明，且只有一套标准，则默认套用自带的 `references/20260320_分类规则完整版.md`。
3. 是否允许在选定标准下新增子分类（遇新情况需请示）
4. 是否需要保留“待确认”区

若用户只给了模糊目标，请先根据选定的标准输出一版建议分类框架，再让用户确认。

### Step 2: 只读盘点，不做改动

先递归盘点目标目录，建立文件清单，再开始内容判断。

优先动作：

- 列出目录结构与文件总量
- 用 `scripts/build_inventory.py` 生成清单 CSV
- 按扩展名、父目录、命名模式做首轮聚类
- 标出空目录、隐藏文件、明显无关文件

若目录很大，先分批扫描，不要一次性读太多文件。

### Step 3: 抽样读内容，建立分类建议

对仅靠文件名无法判断的文件，抽样读取内容；优先读首页、前几页、目录页、表头页。

推荐判断顺序：

1. 先看文件名、父目录、时间、版本号
2. 再看文档首页标题、副标题、作者、日期
3. 再看目录、章节名、表头、结论页
4. 仍不确定时，放入“待确认/边界文件”清单

常用处理方式：

- PDF：提取前几页文本，必要时 OCR
- XMind：解压读取 `content.json` 或等效结构文件
- XLSX：读取 sheet 名、首行字段、样例数据
- DOCX/PPTX：读标题页、目录页、备注区

### Step 4: 先交付建议，不直接开搬

在真正移动前，必须先输出一个“建议结果”，至少包含：

- 已确认可直接移动文件
- 待确认文件
- 待确认-无法稳定读取正文的文件
- 疑似错分文件
- 建议新增或合并的分类
- 需要用户拍板的边界规则

参考 `references/classification-report-template.md` 组织输出。

### Step 5: 得到确认后再执行

只有在用户明确确认后，才执行文件移动。

执行前必须做三件事：

1. 明确提醒批量操作有风险
2. 完整备份目标目录
3. 说明备份位置

若在 Windows 上操作，避免先写带中文路径的 `.ps1` / `.bat` 再执行；优先直接执行命令或使用 Python 逐项移动，避免编码翻车。

### Step 6: 分批移动与逐批核对

移动时遵循小批次原则：

1. 先校验源文件存在、目标文件不存在
2. 再执行移动
3. 再列目录核对结果
4. 再进入下一批

遇到以下情况立即停下：

- 目标已存在同名文件
- 分类规则突然出现冲突
- 抽查后发现本批判断明显不稳

### Step 7: 全量复查

移动完成后做两轮复查：

1. 检查原目录是否还残留业务文件
2. 检查目标目录中是否仍有疑似错分项

复查结果要明确分成三类：

- 已完成，无异常
- 有残留文件待补处理
- 有边界文件需再确认

### Step 8: 删除策略

默认不执行删除。

若用户问“是否可以删掉空目录或备份目录”，先给出：

- 可删除对象清单
- 删除前置条件
- 风险提醒

除非用户再次明确确认，否则停留在建议层；若当前环境对删除权限有限，直接说明并建议用户手动删除。

## Working Rules

### 风险优先级

始终遵循：

- 先只读，后改动
- 先确认，后移动
- 先备份，后执行
- 先核对，后收口

### 分类判断原则

当文件同时像“项目规划”又像“方案分析”时，按“文档主要用途”判定，而不是按零散章节判定。

可采用以下优先顺序：

1. 主要交付目的
2. 文档核心问题意识
3. 主要读者对象
4. 次级内容占比

若仍模糊，直接列入边界清单，不强判。

### 用户沟通输出格式

默认用这 4 段输出：

1. 结论：这一轮建议或执行结果
2. 已确认项：可直接处理内容
3. 待确认项：需要用户拍板的文件或规则
4. 风险/下一步：是否备份、是否执行、是否复查

## Bundled Resources

### `references/20260320_分类规则完整版.md`

泡芙大人独家制定的 0-6 分类法，通过分析文件的核心目的与服务场景进行精准归档，任何无法匹配该大纲的疑似边界文件均需挂起确认。

### `scripts/build_inventory.py`

用于递归生成文件清单 CSV，适合在“只读盘点”阶段快速建立底账。

建议用法：

```bash
python3 scripts/build_inventory.py "/path/to/folder" --output "/tmp/archive_inventory.csv"
```

### `references/workflow-checklist.md`

用于快速复用完整归档流程、风险控制点、边界文件处理方式。

### `references/classification-report-template.md`

用于输出“首轮分类建议”“补漏分析”“总复查结果”这类结构化报告。

## Quick Start

接到任务后，优先按这个顺序行动：

1. 确认目标目录与分类规则
2. 只读盘点并生成 inventory
3. 抽样读内容，形成分类建议
4. 让用户确认边界项
5. 备份后分批移动
6. 递归复查残留与错分
7. 输出最终结果与待删清单
